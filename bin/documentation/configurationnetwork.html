Configuration: Network


File: /configuration/network.cfg

This configuration file can be used to define the network interfaces at which the GCS should receive data. The GCS uses UDP exclusively: both unicast and multicast. The configuration file is a text file and can look like this: 

# ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
# UNICAST
# ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
# Specify a list of UDP unicast sockets at which to receive IMP messages. Format is "PORT@INTERFACE",
# where INTERFACE=0.0.0.0 denotes the default network interface. The key is Network.Unicast.
[Network.Unicast]
    23971@0.0.0.0
    23978@0.0.0.0
    23980@0.0.0.0
    23984@0.0.0.0


# ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
# MULTICAST
# ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
# Specify a list of UDP multicast sockets at which to receive IMP messages. Format is "GROUP:PORT@INTERFACE",
# where INTERFACE=0.0.0.0 denotes the default network interface. The key is Network.Multicast.
[Network.Multicast]
    239.0.0.71:23971@0.0.0.0
    239.0.0.78:23978@0.0.0.0
    239.0.0.80:23980@0.0.0.0
    239.0.0.84:23984@0.0.0.0


# ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
# JOYSTICK DESTINATIONS (UDP-UNICAST)
# ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
# Specify unicast destinations to which to send joystick data. Format is "DESTINATION_IP:DESTINATION_PORT@INTERFACE",
# where INTERFACE=0.0.0.0 denotes the default network interface. The key is JoystickN.Destination with N being
# the zero-based number of the connected joystick.
# [Joystick0.Destination]
#     127.0.0.1:12701@0.0.0.0
The following key values can be set:

Key valueDescription
Network.UnicastSpecifies all unicast interfaces.
Network.MulticastSpecifies all multicast interfaces.
JoystickN.DestinationSpecifies UDP Unicast addresses to which joystick data (from joystick N) should be sent, if connected to the GCS.

Note

While the key-values Network.Unicast and Network.Multicast can be used to specify several adresses in a row, only one destination-address can be given for a joystick.

Unicast

The key value Network.Unicast can contain several associated values. Each value is a string with the following format: 

"PORT@INTERFACE"
Here PORT specifies the UDP port to receive on. INTERFACE specifies the network interface on which messages are to be received. If the computer on which the GCS is running has more than one network interface, you can select the IPv4 address of the desired network interface. If the value "0.0.0.0" is used, the default network interface is used.

Multicast

The key value Network.Multicast can contain several associated values. Each value is a string with the following format: 

"GROUP:PORT@INTERFACE"
Here GROUP specifies the group address. The GCS then joins this group. PORT specifies the UDP port on which to receive data. INTERFACE specifies the network interface on which messages are to be received. If the computer on which the GCS is running has more than one network interface, you can select the IPv4 address of the desired network interface. If the value "0.0.0.0" is used, the default network interface is used.

Joystick

The key value JoystickN.Destination can only contain one associated value. This value is a string with the following format: 

"IP:PORT@INTERFACE"
Here IP specifies the destination address. PORT specifies the destination UDP port. INTERFACE specifies the network interface on which the message is to be sent. If the computer on which the GCS is running has more than one network interface, you can specify the IPv4 address of the desired network interface. If the value "0.0.0.0" is used, the default network interface is used.

As more than one joystick can be connected, different numbers of joystick entries can be added, starting with N = 0. Typically, up to 16 joysticks can be connected. The first joystick that is connected is usually assigned with ID 0. The joystick values (axes and buttons) are read out at the same rate as the graphical user interface is rendered (typ. 60 FPS). As soon as a joystick is connected and a configuration entry is available, the axes and buttons are sent with a specific data protocol via UDP as unicast to the specified address.

Binary Protocol

The Byte-Order corresponds to the Network-Byte-Order (Big-Endian).

ID
(1 byte)		 numAxes
(1 byte)		 numButtons
(1 byte)		 axesData
(numAxes * 4 bytes)		 buttonData
(1 byte per 8 buttons)
Name Data type Description
ID uint8_t Joystick ID (corresponds to N).
numAxes uint8_t Number of axis values for this joystick.
numButtons uint8_t Number of buttons for this joystick.
axesData float[] A float value (big-endian) is transmitted for each axis, which lies in the range [-1.0, 1.0]. Thus, axesData contains numAxes float values.
buttonData uint8[] A bit field, where each bit corresponds to the state of a button (a set bit means "button pressed"). It always starts with the MSBit. The first key is located at position 0x80 of the first byte of buttonData. One byte can contain at most 8 buttons. If the joystick has more than 8 buttons, more bytes are added.

MATLAB Function to decode a joystick message

function [valid, id, numAxes, numButtons, axes, buttons] = DecodeJoystickMessage(bytes, len) % Fallback values valid = false; id = uint8(0); numAxes = uint8(0); numButtons = uint8(0); axes = zeros(256,1); buttons = uint8(zeros(256,1)); % Check minimum length (ID + numAxes + numButtons) if(int32(len) < int32(3)), return; end % Decode header id = uint8(bytes(1)); numAxes = uint8(bytes(2)); numButtons = uint8(bytes(3)); % Check length based on header information N = int32(0); if(numButtons) N = int32(floor((double(numButtons) - 1.0) / 8.0) + 1.0); end if(int32(len) ~= (int32(3) + int32(numAxes)*4 + N)), return; end % Decode all axes index = int32(4); for k = int32(1):int32(numAxes) axes(k) = double(DecodeFloatBigEndian(bytes(index:(index+int32(3))))); index = index + int32(4); end % Decode all buttons shift = int32(-7); for k = int32(1):int32(numButtons) buttons(k) = bitand(uint8(1),bitshift(uint8(bytes(index)),shift)); shift = shift + int32(1); if(shift > 0) index = index + int32(1); shift = int32(-7); end end valid = true; end function out = DecodeFloatBigEndian(in) bytes = typecast(single(-1.0), 'uint8'); isLittleEndian = (uint8(128) == bitand(uint8(128), bytes(4))); out = typecast(uint8([in(4) in(3) in(2) in(1)]), 'single'); if(~isLittleEndian) out = typecast(uint8(in), 'single'); end end
